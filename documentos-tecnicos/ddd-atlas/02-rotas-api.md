## Como ler

- **Metodo**: verbo HTTP (GET, POST, PUT, PATCH, DELETE).
- **Rota DDD**: caminho final canonico (`/api/v1/...`) - este e o contrato.
- **Arquivo**: caminho do arquivo de rota Express.
- **Mount**: prefixo onde o router e montado (`app.use(mount, router)`).
- **Path interno**: path declarado dentro do router (`router.get(path, ...)`).
- **Response Schema**: schema Zod que valida o payload de resposta.
- **Consumidor**: produto/servico que invoca esta rota.
- **Descricao**: o que a rota faz, em uma frase.
- **Alias historico**: rota antes do rename DDD.

Convencoes:
- Apenas valores DDD-finais. Nao mostra estado pre-rename.
- Onde aplicavel, coluna "Alias historico" mostra nome legado (util para grep e git log --follow).
- Linhas marcadas `—` na planilha estao no apendice (nao sao acionaveis).
