01

A Marca

A Gravity é uma plataforma B2B para operações de importação no mercado brasileiro — câmbio, frete, pedidos, licenças, notas fiscais, processos e análise financeira, tudo em um único ecossistema.

Princípio central: A Gravity não é um produto — é um ecossistema. A identidade visual reflete isso: uma marca-mãe forte (Gravity Indigo) que une todas as zonas, com identidades individuais sutis para cada produto.
Conceito do Ícone
Hexágono

Estabilidade e força. Padrão encontrado na natureza (colmeia, basalto). Seis lados = seis produtos principais interconectados.

Ponto Central

O núcleo da plataforma — o ponto de gravidade que atrai e conecta todos os produtos ao redor.

Uso Correto vs Incorreto

Diretrizes obrigatórias para manter a coerência visual em toda a plataforma.

✅ Correto
Cor do produto nos 3 pontos de contato — chip, sidebar dot e item ativo. O resto da UI permanece neutro Solid Slate.
❌ Incorreto
Usar a cor do produto no fundo da página — backgrounds, banners, cabeçalhos coloridos. Quebraria o tema escuro unificado.
✅ Correto
Botão primário sempre em indigo (var(--accent) = #6366f1). Cor da plataforma, não do produto.
❌ Incorreto
Botão primário na cor do produto — um botão verde (#34d399) em SimulaCusto cria inconsistência com o sistema de botões global.
✅ Correto
Logo em cor canônica — usar sempre a cor definida em PRODUTO_META. Importar via @nucleo/logo-produtos.
❌ Incorreto
Recolorir ou distorcer o logo — nunca usar filtro CSS, rotação, cor fora do padrão ou proporção diferente do viewBox 24×24.
✅ Correto
Área de respiro ao redor do logo — mínimo 0.5× a altura do ícone de espaço em todos os lados antes de qualquer outro elemento.
❌ Incorreto
Logo colado a bordas ou outros elementos — sem padding, sem área de respiro, logo recortado ou parcialmente visível.
✅ Correto
Tamanho mínimo: 16px — abaixo disso o ícone perde legibilidade. Para favicons, usar versão simplificada do hexágono.
❌ Incorreto
Logo abaixo de 16px — os detalhes internos do ícone ficam indistinguíveis, prejudicando a identidade visual.

Tokens & Código

Referência técnica para desenvolvedores implementarem a identidade visual corretamente.

CSS Variables (Design Tokens)
/* === Backgrounds === */ --bg-body-dark: #0f172a; --bg-base: #1e293b; --bg-surface: #334155; --bg-elevated: #475569; /* === Acento principal (Indigo 500) === */ --accent: #6366f1; /* botões primários */ --accent-hover: #4f46e5; /* === Texto === */ --text-primary: #f1f5f9; --text-secondary: #94a3b8; --text-muted: #64748b; /* === Status === */ --success: #22c55e; --warning: #f59e0b; --danger: #ef4444;
PRODUTO_META — Fonte única de verdade
// nucleo-global/Logo/produtos/src/produtos.tsx // Import: import { getProdutoMeta } from '@nucleo/logo-produtos' PRODUTO_META = { 'simula-custo': color: '#34d399', // Emerald 400 'pedido': color: '#f59e0b', // Amber 400 'bid-cambio': color: '#06b6d4', // Cyan 500 'bid-frete': color: '#60a5fa', // Blue 400 'lpco': color: '#f43f5e', // Rose 500 'nf-importacao': color: '#c084fc', // Purple 400 'processo': color: '#facc15', // Yellow 400 'financeiro-comex': color: '#f472b6', // Pink 400 }
Importando logos nos produtos
// Logo individual — tamanho e cor via props import { LogoSimulaCusto } from '@nucleo/logo-produtos' <LogoSimulaCusto size={20} color={"#34d399"} /> // no chip <LogoSimulaCusto size={64} /> // no HUB card // Metadados via registry (cor + ícone + sublabel) import { getProdutoMeta } from '@nucleo/logo-produtos' const meta = getProdutoMeta('simula-custo') meta.color // '#34d399' meta.icon // <LogoSimulaCusto size={16} /> meta.sublabel // 'fiscal · NCM'
